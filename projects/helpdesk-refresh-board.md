# HelpDesk Refresh Board README

HelpDesk Refresh Board は、HelpDesk Motor Company FAQ 刷新ワークショップで進行状況を共有するための GitHub Projects (ボード) です。参加者が Issue を作成してから FAQ 公開までの流れを「未着手 → 作業中 → レビュー → 公開済み」で可視化し、2 時間のハンズオンでも迷わずタスクを進められるように設計しています。

## 1. ボード概要

- **用途**: FAQ 更新タスクの進捗管理 / SLA 可視化 / レビュー待ちの早期発見
- **推奨ビュー**: Board ビューをデフォルトに設定し、列移動だけで状況把握ができるようにする
- **参考構成ファイル**: `projects/helpdesk-refresh.json`（列・カスタムフィールド・サンプルカード定義）

### 列（Columns）

| 列名 | 目的 |
| --- | --- |
| 未着手 | 受信しただけの問い合わせ。Issue 作成直後の置き場所 |
| 作業中 | ブランチを切って FAQ を編集中 |
| レビュー | Pull Request を作成しレビュー待ち |
| 公開済み | main にマージし、FAQ と README が更新済み |

### カスタムフィールド

| フィールド | 種別 | 詳細 |
| --- | --- | --- |
| 優先度 | Single select | High / Medium / Low。SLA の共有に使用 |
| 期限 | Date | 対応期限。ワークショップ中は当日付で問題ありません |
| Slack 通知要否 | Single select | 「必要 / 不要」。運営 Slack 連携用 |

## 2. 新規作成手順

1. GitHub で対象リポジトリを開き、`Projects > New project` をクリック。
2. `Board` テンプレートを選択し、名前を **HelpDesk Refresh Board** に設定。
3. 右上の `...` メニューから `Manage workflow` を開き、以下を設定。
   - 列: 「未着手 / 作業中 / レビュー / 公開済み」を順番に追加
   - カスタムフィールド: 優先度（単一選択）、期限（日付）、Slack 通知要否（単一選択）
4. `projects/helpdesk-refresh.json` の値を参考に、サンプルカード（Issue #101〜#103）を必要に応じて手動で追加。
5. ワークショップ参加者へ共有する場合は、`Share` ボタンからアクセス権（`Can edit` もしくは `Can view`）を付与。

## 3. 参加者向け運用ヒント

- Issue を作成したら、右ペインの `Projects` でこのボードを選択し、列「未着手」にカードが生成されているか確認します。フォークや個人リポジトリでは自動表示されないため、事前に主催者からアクセス権を配布してください。
- Pull Request を作成したタイミングでカードを「レビュー」に移動し、マージ後は「公開済み」へ移します。列を移すだけで進捗共有が完了するため、会場の大画面にボードを投影して SLA メーターとして活用できます。
- 優先度や期限を更新した際は、Slack 通知要否フィールドを利用して `必要` に切り替えると、運営チームが迅速にサポートできます。

## 4. トラブルシュート

| 症状 | 対処 |
| --- | --- |
| Projects 一覧にボードが出てこない | ボード所有組織のメンバーであるか確認し、アクセス権を付与してもらう。フォークでは自動コピーされません |
| 列を追加できない | 権限が `Can view` の場合は編集できません。主催者に `Can edit` 権限を依頼 |
| カスタムフィールドが表示されない | `+ Add field` でフィールドを列に表示する設定を有効化 |
