# WiFi に接続できない場合の基本手順

オフィスの WiFi（Corp SSID）に接続しようとしても認証画面が表示されず、接続できない場合の確認方法と対処方法です。各デバイスごとの注意点もまとめています。

## 症状別のヒント

- SSID「Corp」を選択しても認証画面が表示されない
- 認証情報を入力する画面が出ない／接続が完了しない
- 他の WiFi には接続できるが、Corp だけできない

## すぐに試せること（3 分）

1. **SSID の選択を再確認**
    - SSID「Corp」が正しく選択されているか確認します。類似の名前（Corp-Guest など）と間違えやすいので注意。
2. **WiFi を一度 OFF/ON**
    - WiFi を一度 OFF にしてから再度 ON にし、再接続を試みます。
3. **端末の再起動**
    - PC やスマートフォンを再起動してから再度 SSID を選択します。
4. **他の端末でテスト**
    - 同僚の端末で同じ SSID に接続できるか確認します。

## PC・モバイル側のチェック

1. **ネットワーク設定のリセット**
    - Windows: 「設定」→「ネットワークとインターネット」→「ネットワークのリセット」
    - Mac: 「システム設定」→「Wi-Fi」→「ネットワークを削除」→再追加
    - iOS/Android: WiFi設定から「Corp」ネットワークを削除し、再度追加
2. **認証方式の確認**
    - Corp SSID は「WPA2-Enterprise」や「802.1X」認証が必要です。認証方式が「パスワードのみ」になっていないか確認。
3. **証明書のインストール**
    - 会社指定の証明書が必要な場合は、IT部門の案内に従いインストールしてください。
4. **OS・ドライバーの更新**
    - Windows/Mac の場合は WiFi ドライバーや OS を最新に更新します。

## デバイス別の注意点

### Windows PC
- 「デバイスとプリンター」や「ネットワークと共有センター」から WiFi 設定を確認。
- 企業ネットワークの場合、プロファイルの自動構成が必要な場合があります。
- 「証明書のインストール」や「グループポリシー」の影響で認証画面が出ないことがあります。

### Mac
- 「システム設定」→「Wi-Fi」→「詳細」からネットワークを削除し再追加。
- キーチェーンアクセスで古い証明書が残っている場合は削除。

### iOS / Android
- ネットワークを一度削除し、再度 SSID を選択。
- 企業 WiFi の場合、証明書やプロファイルのインストールが必要なことがあります。

## それでも接続できないとき

- 端末の WiFi 設定画面やエラーコードをスクリーンショットで保存し、Issue テンプレートの「添付情報」にアップロードしてください。
- どの画面で止まっているか、認証画面が出ない場合は「SSID選択後の画面」を添付すると伝わりやすいです。
- 急ぎの場合は、モバイルデータや別の WiFi（Corp-Guest など）を一時的に利用してください。
