# Copilot プロンプト例集

GitHub Copilot / Copilot Chat を FAQ 更新に活用するための具体例をまとめました。文章生成だけでなく、レビューや改善提案にも使えます。

## 1. FAQ の下書きを作る

```text
以下の入力を読み、HelpDesk Motor Company の FAQ 形式（見出し + 箇条書き）で下書きを作成してください。
- 問題: VPN でエラー 619 が出て接続できない
- 想定読者: IT に詳しくない社内メンバー
- トーン: 丁寧で安心感のある文体
- 必ずステップを 3 つ以上に分ける
```

### ポイント

- 「読みやすさ」「対象読者」を必ず書くと、専門用語だらけになることを防げます。
- 箇条書きの数や文章量も条件に含めると調整しやすいです。

## 2. 文章をやさしく書き換えてもらう

```text
次の文章を IT 初心者にも伝わるように言い換えてください。略語にはカッコ書きで説明を追加してください。
<貼り付けたいテキスト>
```

- Copilot Chat に貼り付ける量が多いときは、段落ごとに分割すると精度が上がります。

## 3. レビューコメントを考えてもらう

```text
Pull Request の差分を読み、次の観点でレビューコメントを作成してください。
- 手順が飛んでいないか
- 用語の説明が不足していないか
- 読み手が迷いそうな箇所があれば改善案を添える
差分:
<PR 差分を貼り付け>
```

- 実際の PR では Copilot Code Review を有効化すると、自動で似た観点のコメントを提示してくれます。

## 4. Issue 返信文のドラフトを依頼する

```text
Issue の内容に対して、ヘルプデスク担当者として返信してください。
- 迅速なお礼で始める
- 具体的に確認してほしいスクリーンショットを 2 点示す
- テンプレの「次のアクション」に貼り付けられる形式でまとめる
Issue:
<Issue 本文>
```

## 5. CLI の補足説明を生成する

```text
次の CLI コマンドが初心者にとって難しそうなので、1 行説明を添えてください。
```

```text
gh issue create --title "VPN" --body "..."
```

## 6. プロンプトを書くときのチェックリスト

- 目的（下書き / 校正 / 返信 など）を 1 行で宣言する。
- 読者像（初心者 / 情シス / 管理者）を書く。
- 出力フォーマット（箇条書き / 表 / Markdown）を指定する。
- 必要なら禁止事項（略語は使わない 等）を添える。

迷ったときはこのページを参照し、カスタマイズしながら活用してください。
